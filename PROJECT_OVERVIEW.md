# Документация и Карта Проекта: AppoiNtmenK

## 🛠 Технологический стек
- **Frontend (Пользовательский интерфейс):** Angular (TypeScript) - современный и мощный фреймворк для создания одностраничных веб-приложений (SPA).
- **Backend (Серверная часть):** Django (Python) + Django REST Framework - фреймворк для создания надежных API и серверной бизнес-логики.
- **База данных:** SQLite3 (`db.sqlite3`) - легковесная реляционная база данных (используется по умолчанию в Django, отличный выбор для разработки и прототипирования).

---

## 🗺 Карта проекта (Архитектура)

Проект четко разделен на независимые клиентскую и серверную части:

### 1. Backend (Сервер - Каталог `/backend/`)
Отвечает за работу с данными, безопасность, обработку запросов и интеграции.

- `manage.py` — главная утилита командной строки Django (запуск сервера, миграции, создание админов).
- `db.sqlite3` — сам файл базы данных.
- **`appointmenk/`** — ядро и конфигурация проекта:
  - `settings.py` — глобальные настройки сервера (БД, безопасность, JWT токены, CORS).
  - `urls.py` — корневой маршрутизатор API (роутер).
  - `asgi.py` / `wsgi.py` — интерфейсы для развертывания приложения и поддержки асинхронных соединений (например, для WebSockets).
- **`core/`** — главное приложение с бизнес-логикой:
  - `models.py` — описание таблиц БД (User, SpecialistDetails, Appointment, ChatMessage и Availability).
  - `views.py` — контроллеры (API Endpoints). Принимают запросы от сайта, обрабатывают их и формируют ответы.
  - `serializers.py` — "переводчик" между сложными объектами Python/БД и форматом JSON, который понимает Angular.
  - `urls.py` — локальные пути конкретно для компонента core (например, маршруты пользователей и записей).
  - `google_calendar.py` — модуль интеграции. Синхронизирует записи на приём с реальным Google Календарем.
  - `migrations/` — история изменения структуры базы данных (для создания и модификации таблиц).

### 2. Frontend (Клиент - Каталог `/frontend/`)
Отвечает за отображение данных, интерактивность и логику на стороне браузера пользователя.

- `package.json` — список JS/TS библиотек и зависимостей фронтенда.
- `angular.json` — конфигурация компилятора Angular.
- **`src/app/`** — исходный код приложения:
  - **`core/`** — ядро клиента (системные модули):
    - `services/` (`auth`, `appointment`, `chat` и др.) — классы, которые делают HTTP-запросы к Backend'у.
    - `guards/` (`auth.guard.ts`) — "охранники" маршрутов. Проверяют авторизован ли пользователь, чтобы пустить его на защищенные страницы.
    - `interceptors/` (`auth.interceptor.ts`) — перехватывают все запросы на бекенд и автоматически добавляют токен безопасности (JWT).
  - **`features/`** — страницы и их логика:
    - `auth/` — окна регистрации и входа в систему.
    - `booking/` & `availability-calendar/` — интерфейс для просмотра свободных слотов и записи на приём к специалистам.
    - `dashboard/` & `specialist-dashboard/` — личные кабинеты (один для клиентов, другой для самих специалистов).
    - `chat/` & `chat-list/` — система обмена сообщениями между врачом/специалистом и клиентом.
  - **`shared/`** — переиспользуемые элементы интерфейса (как `navbar` - верхнее меню, или `status-badge` - бейджики статусов).

## 💡 Как работают основные процессы

1. **Запись на прием (Бронирование)**
   * **Frontend:** Пользователь открывает компонент `booking.component.ts`. Срабатывает скрипт, который через `appointment.service.ts` шлёт HTTP-запрос к API.
   * **Backend:** Запрос попадает в `urls.py`, затем в метод внутри `views.py`. Метод обращается к базе данных через модели (`models.py`) и возвращает список свободных слотов.
   * Когда пользователь выбирает время, Frontend отправляет `POST` запрос. Backend сохраняет запись в SQLite и, если подключена интеграция, отправляет событие через `google_calendar.py`.

2. **Безопасность и Авторизация**
   * Когда пользователь вводит логин и пароль в приложении Angular, запрос летит на бекенд.
   * В случае успеха, бекенд отдает зашифрованный ключ (Токен Авторизации).
   * Фронтенд сохраняет ключ. Во время всех будущих действий (написать в чат, открыть дашборд), файл `auth.interceptor.ts` незаметно прикрепляет этот ключ к запросам. Django видит ключ и понимает, кто именно делает запрос.

3. **Общение (Чаты)**
   * Сервис `chat.service.ts` связывается с бекендом для вывода истории (`ChatMessage`). Модель бекенда хранит ссылки на отправителя и получателя сообщения и сам текст.
